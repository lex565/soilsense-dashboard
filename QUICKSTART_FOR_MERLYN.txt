================================================================================
  THE 4 LAYERS EXPLAINED
================================================================================

LAYER 1: Data Input
  What: Fetch SMAP, Sentinel-2, CHIRPS, u-blox daily
  Why: Multiple data sources improve predictions
  Output: Raw data ready for processing

LAYER 2: Processing & Modeling
  What: Random Forest + Kriging downscale 9-km SMAP → 10-m maps
  Why: 10-m resolution enables field-level irrigation decisions
  Output: 10-m soil moisture maps + alerts (SM < 0.10 critical, < 0.15 warning)

LAYER 3: Validation & Quality Control
  What: Compare predictions vs. independent u-blox GNSS-R ground truth
  Why: Validation metrics (Table 19) build farmer trust
  Output: Trust Score 🟢 (all metrics pass) = high confidence
  Metrics:
    - Pearson R: 0.93 (target ≥0.80) ✓
    - RMSE: 0.0181 m³/m³ (target ≤0.040) ✓
    - MAE: 0.0131 m³/m³ (target ≤0.030) ✓
    - Bias: +0.0021 m³/m³ (target ±0.02) ✓

LAYER 4: Output Channels
  What: Deliver alerts to farmers (SMS + web + mobile + CSV)
  Why: Farmers need multiple access methods (no internet in field)
  Output:
    - SMS: bilingual, 160 chars, via Twilio/Afrimotech
    - Web: 10-m map + validation metrics + alert log
    - Mobile: offline-first Flutter app
    - CSV: for DALRRD government reports
